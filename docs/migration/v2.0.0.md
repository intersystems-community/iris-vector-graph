# Migration Guide: v2.0.0

## TL;DR — No breaking changes for library users

The v2.0.0 internal refactor (spec 186) is **API-compatible**. If you use
`iris_vector_graph` as a library, **no code changes are required**.

- `from iris_vector_graph import IRISGraphEngine` — unchanged
- `from iris_vector_graph.engine import IRISGraphEngine` — unchanged
- All 162 public methods of `IRISGraphEngine` keep identical signatures
- `IRISGraphEngine.TableNotMappedError` — still reachable
- Cypher / AQL / GraphQL surfaces — unchanged

A regression guard (`tests/regression/test_engine_api.py`) snapshots the full
public method set and signatures and fails CI on any accidental change.

## What changed internally (for contributors)

### `engine.py` decomposed into domain mixins

`engine.py` shrank from **8,073 lines to 846**. `IRISGraphEngine` is now a thin
facade composing ten domain mixins under `iris_vector_graph/_engine/`:

| Mixin | Module | Responsibility |
|---|---|---|
| `TemporalMixin` | `_engine/temporal.py` | temporal edges, windows, bucket aggregates |
| `SnapshotMixin` | `_engine/snapshot.py` | import/export (RDF, NetworkX, NDJSON, OBO), snapshots |
| `FhirMixin` | `_engine/fhir.py` | SQL table mapping, FHIR bridge |
| `AdminMixin` | `_engine/admin.py` | status, query lifecycle, SHOW commands, warnings |
| `EmbeddingsMixin` | `_engine/embeddings.py` | embedding storage/retrieval, async queue |
| `SchemaMixin` | `_engine/schema.py` | schema deploy, introspection, inference, ^KG/^NKG sync |
| `NodesEdgesMixin` | `_engine/nodes_edges.py` | node/edge CRUD, bulk ops, named graphs |
| `QueryMixin` | `_engine/query.py` | Cypher/AQL execution, var-length/shortest-path routing |
| `AlgorithmsMixin` | `_engine/algorithms.py` | centrality, community, PPR, traversal operators |
| `VectorMixin` | `_engine/vector.py` | VecIdx/IVF/PLAID/BM25, vector & hybrid search |

`engine.py` retains only connection plumbing (`__init__`, `from_connect`,
`from_wrapper`, reconnect/arno helpers, `_iris_obj`) and the system-procedure
registry.

The `_engine` package is **private** (underscore-prefixed). Do not import mixins
directly; always go through `IRISGraphEngine`.

### System-procedure dispatch is now a registry

`_try_system_procedure` (a 478-line `if name == ...` chain) is now a
`_SYSTEM_PROCEDURES` dict mapping procedure name → `_proc_*` handler method.
Prefix-matched procedures (`apoc.*`, `db.*`, `dbms.*`) remain in a trailing
if-chain. Behavior is identical.

### Cypher translator decomposed (no API change)

`cypher/translator.py` internals were decomposed so that **every function is
now cyclomatic complexity ≤ 25** (the worst was `translate_expression` at 322).
The public translation entry points are unchanged.

## If you were importing private internals

If you reached into `iris_vector_graph.engine` for a **private** method
(`_`-prefixed) that moved to a mixin, it still resolves on an `IRISGraphEngine`
*instance* via inheritance, but is no longer a module-level attribute of
`engine.py`. Private internals were never part of the supported API; switch to
the public method or open an issue if you have a real need.
