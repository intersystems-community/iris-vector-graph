# Data Model: initialize_schema() Stored Procedure Installation

**Feature**: 020-initialize-schema-stored-procedures  
**Date**: 2026-02-27

---

## Overview

This feature has **no new data entities**. It is a bug fix to existing schema installation logic. The data model below documents the affected code entities (classes, methods, their inputs/outputs) and the state transitions that are being fixed.

---

## Code Entities

### `GraphSchema` (iris_vector_graph/schema.py)

**Current state**: Module has a `SyntaxError` — unimportable.  
**Target state**: Module is syntactically valid; `get_procedures_sql_list` has a single, correct implementation.

#### `GraphSchema.get_procedures_sql_list(table_schema, embedding_dimension)`

| Field | Type | Description |
|-------|------|-------------|
| `table_schema` | `str` | SQL schema name for data tables (e.g. `"Graph_KG"`) |
| `embedding_dimension` | `int` | Vector dimension used in the `DECLARE qvec VECTOR(DOUBLE, N)` clause |
| **returns** | `List[str]` | Ordered list of SQL DDL statements to execute |

**Returned statements (in order)**:
1. `CREATE SCHEMA iris_vector_graph` — idempotent (caller wraps in try/except for "already exists")
2. `CREATE OR REPLACE PROCEDURE iris_vector_graph.kg_KNN_VEC(...)` — vector KNN search
3. `CREATE OR REPLACE PROCEDURE iris_vector_graph.kg_TXT(...)` — full-text search
4. `CREATE OR REPLACE PROCEDURE iris_vector_graph.kg_RRF_FUSE(...)` — RRF hybrid fusion

**Signature change** (backward-incompatible internal change; no public API impact):  
Before: `get_procedures_sql_list(table_schema="Graph_KG") -> List[str]`  
After: `get_procedures_sql_list(table_schema="Graph_KG", embedding_dimension: int = 1000) -> List[str]`

The default `1000` preserves existing behavior for any caller that doesn't pass the dimension explicitly, while `initialize_schema()` always passes the real dimension.

---

#### `iris_vector_graph.kg_KNN_VEC` (IRIS stored procedure)

This is the SQL procedure installed in IRIS. Its DDL is generated by `get_procedures_sql_list`.

| Parameter | SQL Type | Description |
|-----------|----------|-------------|
| `queryInput` | `VARCHAR(32000)` | JSON float array (pre-computed vector) OR raw text (if `embeddingConfig` set) |
| `k` | `INT DEFAULT 50` | Number of nearest neighbours to return |
| `labelFilter` | `VARCHAR(128) DEFAULT NULL` | Optional node label to restrict search |
| `embeddingConfig` | `VARCHAR(128) DEFAULT NULL` | Optional IRIS `%Embedding.Config` name for server-side vectorization |
| **returns** | `TABLE(id VARCHAR(256), score DOUBLE)` | Ordered result set, `score` descending |

**Internal state**:
- `DECLARE qvec VECTOR(DOUBLE, {embedding_dimension})` — dimension fixed at install time, must match `kg_NodeEmbeddings.emb` column dimension
- Uses `TO_VECTOR(queryInput)` when `embeddingConfig` is empty, `EMBEDDING(queryInput, embeddingConfig)` otherwise
- `SELECT TOP (k) ... ORDER BY VECTOR_COSINE(n.emb, qvec) DESC` — IRIS HNSW index is utilized when `TOP + ORDER BY VECTOR_* DESC` pattern is present

### `IRISGraphEngine` (iris_vector_graph/engine.py)

#### `IRISGraphEngine.initialize_schema()`

**State machine** (current → target):

```
Current:
  [validate dim] → [create Graph_KG schema] → [create tables] →
  [ensure indexes] → [check dim mismatch] →
  [install procedures: silent failure] → [commit]

Target:
  [validate dim] → [create Graph_KG schema] → [create tables] →
  [ensure indexes] → [check dim mismatch] →
  [install procedures: collect failures] →
    ├─ failures? → raise RuntimeError (do NOT commit)
    └─ no failures → [commit]
```

**Error classification**:

| IRIS Error substring | Classification | Action |
|----------------------|----------------|--------|
| `"already exists"` | Expected (idempotent re-run) | Silently ignored |
| `"already has"` | Expected (idempotent re-run) | Silently ignored |
| anything else | Unexpected DDL failure | Collected → `RuntimeError` after loop |

**RuntimeError message format**:
```
initialize_schema() failed to install N stored procedure(s). 
Server-side vector search will be unavailable. 
First error: <underlying DB error message>
```

---

### Procedure-Calling Side: `IRISGraphEngine.kg_KNN_VEC()`

No change to the method signature or behavior. The fix is upstream (installation). After this fix:
- The `WARNING: Server-side kg_KNN_VEC failed` message should never appear on a properly initialized schema
- The Python fallback (`_kg_KNN_VEC_python_optimized`) remains as a safety net for unexpected runtime failures but is no longer the default path

## State Transitions

### Schema initialization lifecycle (post-fix)

```
Fresh install:
  initialize_schema(embedding_dimension=384)
    │
    ├─ CREATE SCHEMA Graph_KG          [ok]
    ├─ CREATE TABLE nodes              [ok]
    ├─ CREATE TABLE rdf_labels         [ok]
    ├─ CREATE TABLE rdf_props          [ok]
    ├─ CREATE TABLE rdf_edges          [ok]
    ├─ CREATE TABLE kg_NodeEmbeddings  [ok: VECTOR(DOUBLE,384)]
    ├─ CREATE SCHEMA iris_vector_graph [ok]
    ├─ CREATE OR REPLACE PROC kg_KNN_VEC  [ok: VECTOR(DOUBLE,384)]
    ├─ CREATE OR REPLACE PROC kg_TXT      [ok]
    ├─ CREATE OR REPLACE PROC kg_RRF_FUSE [ok]
    └─ commit

Second call (idempotent):
  initialize_schema(embedding_dimension=384)
    │
    ├─ CREATE SCHEMA Graph_KG          [ignored: already exists]
    ├─ CREATE TABLE nodes              [ignored: already exists]
    ├─ ...
    ├─ CREATE SCHEMA iris_vector_graph [ignored: already exists]
    ├─ CREATE OR REPLACE PROC kg_KNN_VEC  [ok: replaces existing]
    └─ commit
```

## Validation Rules

- `embedding_dimension` MUST be a positive integer before `initialize_schema()` is called (enforced by existing guard: raises `ValueError` if `dim is None`)
- `embedding_dimension` MUST match the dimension of the `kg_NodeEmbeddings.emb` column (mismatch logged at `ERROR` level by existing dimension-check logic; the new procedure will use the engine's configured dimension)
- SQL statements with empty/whitespace content MUST be skipped (existing `if not stmt.strip(): continue` guard)
