# Spec 093: Arno Acceleration Benchmark Suite

**Feature Branch**: `093-arno-acceleration-benchmark`
**Created**: 2026-05-03
**Status**: Draft
**Cross-reference**: Spec 079 (Rust-Accelerated BFS), Spec 028 (^NKG Integer Index)

## Goal

A **repeatable, self-contained benchmark harness** that measures IVG traversal performance
across four execution paths:

| Path | Label |
|------|-------|
| ObjectScript `BFSFastJson` over `^KG` | `ivg-os` |
| Rust `NKGAccel.BFSJson` over `^NKG` (arno) | `ivg-arno` |
| Neo4j Cypher `MATCH (n)-[:R*1..D]->(m)` | `neo4j` |
| Memgraph Cypher `MATCH (n)-[:R*1..D]->(m)` | `memgraph` |

The harness must be runnable at any time with a single command, produce a structured JSON
result file, and print a human-readable comparison table. Results are reproducible: same
graph seed → same results.

---

## Motivation

Spec 079 shipped arno BFS with a projected speedup of 128ms → <30ms (p50, 10K/50K graph).
That projection was never validated with a measured benchmark. We also have no head-to-head
comparison of IVG vs Neo4j vs Memgraph on the same graph and same query patterns.

External benchmarks (mgBench, LSQB, graph-bench) use their own datasets and measure only
vendor systems. We need IVG numbers on the same scale to:

1. Validate arno actually delivers the projected speedup
2. Establish where IVG sits relative to Neo4j/Memgraph for each query shape
3. Identify the hop depth at which arno diverges from BFSFastJson (the crossover point)
4. Provide publishable performance claims with methodology attached

---

## Query Catalog

The benchmark covers six query shapes, matching patterns from external benchmarks:

| ID | Query Shape | openCypher | IVG Engine Path | Arno path | External analog |
|----|------------|------------|-----------------|-----------|-----------------|
| Q1 | 1-hop expand, count | `MATCH (s)-[:R]->(n) RETURN count(n)` | MatchEdges → SQL | none (no BFS) | mgBench expansion_1 |
| Q2 | 2-hop expand, DISTINCT | `MATCH (s)-[:R*1..2]->(n) RETURN DISTINCT n.id` | BFSFastJson d=2 | NKGAccel.BFSJson d=2 | mgBench expansion_2 |
| Q3 | 3-hop expand, DISTINCT | `MATCH (s)-[:R*1..3]->(n) RETURN DISTINCT n.id` | BFSFastJson d=3 | NKGAccel.BFSJson d=3 | mgBench expansion_3 |
| Q4 | 4-hop expand, DISTINCT | `MATCH (s)-[:R*1..4]->(n) RETURN DISTINCT n.id` | BFSFastJson d=4 | NKGAccel.BFSJson d=4 | mgBench expansion_4 |
| Q5 | shortestPath (BFS) | `MATCH p=shortestPath((a)-[*..8]-(b)) RETURN length(p)` | ShortestPathJson | NKGAccel (future) | mgBench IC13 (0.21ms) |
| Q6 | Weighted shortest path | `CALL ivg.shortestPath.weighted($a,$b,'weight',9999,10) YIELD totalCost` | Dijkstra ObjectScript | none yet | n/a |

**Note**: Q1 does not route through BFS/arno — it is included as the 1-hop baseline to
establish where the ObjectScript `$Order` path is already competitive.

## Datasets

Two synthetic graphs generated by `graph_gen.py` (existing `RMATGenerator`), plus an
optional real-data path using the Mindwalk NCIT graph loaded in `gqs-ivg-test`.

| Dataset | Nodes | Edges | Avg degree | Purpose |
|---------|-------|-------|------------|---------|
| **S** (small) | 1,000 | 5,000 | 5.0 | Warm-up, sanity check |
| **M** (medium) | 10,000 | 50,000 | 5.0 | Primary target — matches spec 079 baseline |
| **L** (large) | 100,000 | 500,000 | 5.0 | Scale test — arno advantage should grow |
| **R** (real, optional) | ~447K | ~1.5M | ~3.4 | Mindwalk NCIT+HLA graph (string IDs, sparser) |

All synthetic graphs use the same RMAT parameters (`a=0.57, b=0.19, c=0.19, d=0.05`) and
the same random seed (`42`) for reproducibility. The graph is regenerated from seed each
run — no persistent CSV dependency.

## Benchmark Protocol

### Warmup
- 3 warmup runs discarded before measurement
- IVG: `BuildKG()` + `BuildNKG()` called once before any query
- `^KG` and `^NKG` populated and verified before timed runs

### Measurement
- **10 timed runs** per (dataset, query, path) combination
- Record: min, p50, p90, p99, max, result_count
- Cold vs hot distinction: first timed run = cold, runs 2–10 = hot; report both p50s

### Metrics
- **Latency (ms)**: wall-clock from query dispatch to result receipt
- **TEPS**: Traversal Edges Per Second = `result_count / latency_s`
- **Result count**: number of distinct nodes/paths returned (must match across IVG paths)
- **Correctness check**: `ivg-os` vs `ivg-arno` result sets must be identical (set equality)

### External system runs (optional)
Neo4j and Memgraph runs require running containers. The harness detects presence and skips
gracefully if unavailable. Connection params are configurable via environment variables.

## Usage

### Minimal run (IVG only, medium dataset, no external systems)
```bash
cd /Users/tdyar/ws/iris-vector-graph
python tests/benchmarks/bench.py
```

### Full run (all datasets, with Neo4j and Memgraph)
```bash
python tests/benchmarks/bench.py \
  --datasets S M L \
  --neo4j-uri bolt://localhost:7687 \
  --neo4j-password neo4j \
  --memgraph-uri bolt://localhost:7687 \
  --runs 10
```

### Real data run (Mindwalk NCIT graph already loaded in gqs-ivg-test)
```bash
python tests/benchmarks/bench.py \
  --dataset R \
  --iris-host localhost --iris-port 1972
```

### Re-run with saved graph (skip data load)
```bash
python tests/benchmarks/bench.py --skip-load --datasets M
```

### Compare two result files
```bash
python tests/benchmarks/bench.py --compare \
  results/bench_20260501_120000.json \
  results/bench_20260503_143200.json
```

### Environment variables (override CLI flags)
```bash
IRIS_HOST=localhost IRIS_PORT=1972 IRIS_NAMESPACE=USER \
IRIS_USERNAME=_SYSTEM IRIS_PASSWORD=SYS \
NEO4J_URI=bolt://localhost:7687 NEO4J_PASSWORD=neo4j \
MEMGRAPH_URI=bolt://localhost:7688 \
python tests/benchmarks/bench.py
```

## Acceptance Criteria

### Functional
- **SC-001**: `python tests/benchmarks/bench.py` runs end-to-end with no external deps (IVG only)
- **SC-002**: Produces valid JSON result file in `tests/benchmarks/results/`
- **SC-003**: Prints comparison table to stdout
- **SC-004**: Correctness check fails loudly if `ivg-os` and `ivg-arno` result sets differ
- **SC-005**: `--skip-load` works correctly — reuses existing `^KG`/`^NKG` without rebuild
- **SC-006**: Missing Neo4j/Memgraph emits a warning and skips gracefully (no crash)
- **SC-007**: `--compare` mode prints a delta table between two result files

### Performance targets (dataset M, hot p50)
- **SC-008**: `ivg-arno Q2` ≤ 5ms (2-hop, arno path) — **BLOCKED: requires spec 094**
- **SC-009**: `ivg-arno Q3` ≤ 30ms (3-hop, arno path — spec 079 target) — **BLOCKED: requires spec 094**
- **SC-010**: `ivg-arno Q4` ≤ 60ms (4-hop, arno path) — **BLOCKED: requires spec 094**
- **SC-011**: `ivg-os Q1` ≤ 1ms (1-hop baseline — should already pass)
- **SC-012**: `ivg-arno` result count == `ivg-os` result count for Q2/Q3/Q4 (correctness)

### Repeatability
- **SC-013**: Two consecutive runs on an idle container produce p50 values within ±15%
- **SC-014**: Same graph seed (`--seed 42`) always produces the same graph structure

## Implementation Notes

### Graph generation
Use existing `RMATGenerator` from `graph_gen.py`. Add `seed` parameter if not already
present so runs are reproducible. Write edges to `^KG` directly via `BuildKG()` — do not
go through `rdf_edges` SQL for the benchmark (avoids SQL ingest time from the measurement).

### Arno detection
Use `engine.status().arno_available` (from spec 080 `EngineStatus`). If `False`, skip
`ivg-arno` path and emit a warning. The benchmark is still useful for `ivg-os` vs
Neo4j/Memgraph comparison.

### Seed node selection
Use the highest-degree node in the graph as the BFS seed — this produces the largest,
most interesting result set and is the worst case for both IVG and competitors. Degree
computed via `^KG("deg", node)` before measurement starts.

### Neo4j / Memgraph load
Use existing `load_neo4j.py` from `tests/benchmarks/`. Load the same RMAT-generated edge
list into Neo4j/Memgraph before running external queries. Add equivalent `load_memgraph.py`
if it does not exist.

### Q5 (shortestPath) seed pair selection
Pick two nodes at graph distance ≥4 (guaranteed by construction: start from seed,
take the node at exactly BFS step 4). This ensures the shortest path is non-trivial.

### Q6 (weighted shortest path)
Set edge weights from the RMAT generator edge weight field (`w` column). If RMAT does not
generate weights, assign `weight = 1.0 + random(0, 4)` per edge at load time.

## Out of Scope

- PPR benchmarking — covered separately when arno PPR path is validated (future spec)
- LSQB multi-hop pattern queries (Q1–Q9) — require LDBC SNB dataset ingest (separate effort)
- WCC / CDLP benchmarking — graph-wide algorithms, different from traversal benchmarks
- `allShortestPaths` at scale — future work, depends on arno path for `ShortestPathJson`
- Continuous / CI benchmark runs — out of scope; this is a manual run-on-demand harness
